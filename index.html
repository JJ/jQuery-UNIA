Tutorial de jQuery y jQuery Mobile

Por JJ Merelo

Curso Desarrollo de aplicaciones multiplataforma para dispositivos móviles, UNIA, La Rábida, Huelva, julio 2013

Qué es jQuery

  • Librería de JavaScript
  • Simplifica la creación de programas e interfaces ricos de usuario.
  • Creado en el año 2006.
  • Usado en la mitad de los sitios más populares (más que Flash).
  • La mejor librería para manipulación del DOM, Ajax y efectos visuales.

Software libre, por supuesto

  • Usa una licencia MIT
  • La licencia MIT es permisiva y permite incluirlo dentro de software privativo
  • Eso no significa que debas hacerlo.

Donde conseguirlo

  • Descargable desde jQuery.org.
  • Y usable desde ahí o desde Google hosted libraries o cualquier otra Content Delivery Network (CDN).
  • Uso minificado: descarga rápida.
  • O, por supuesto, desde tu propia copia en tu propio sitio web (porque es libre).
  • Abstrae las diferentes implementaciones de los navegadores.

    • Base de un ecosistema de módulos

    • jQuery UI, para creación de interfaces de usuario.
    • jQuery mobile, para móviles y basada en HTML5
    • Y plugins para jQuery
    • Aparte de jmpress.js, con el que estoy haciendo esta presentación.

    Claves del programa anterior

    • Inclusión del jQuery alojado por Google.
    • $ es un identificador válido de JavaScript y una abreviatura de jQuery
    • $(document) == jquery(document) llama a una función sobre el objeto document.
    • ... que devuelve otro objeto sobre el que creamos un evento cuando el documento está ready.

    Creando una agenda

    jQuery crea eventos: cuando suceda esto (ready), haz esto otro (alert): estilo asíncrono

    Simplificando aún más

    Tan simple, tan simple...

    • jQuery usa extensivamente las funciones anónimas o closures de JavaScript
    • Hemos creado un evento que se lanza cuando el documento está cargado
    • Se crea una función anónima a la que se llama asíncronamente

    Recuerda, jQuery es siempre asíncrono

    Seleccionando partes del documento

    • Se usan selectores CSS3 entre comillas: $("h2"), por ejemplo.
    • Un selector devuelve un set jQuery. un objeto sobre el cual se puede aplicar otra función: 
    $("h2").each( function() { 
    alert(this.text);
}

    Ejemplo: Cambiar color a rojo

    Unas palabrejas cualesquiera

    $("#a_rojo form").submit(function(){
    $("#a_rojo").css( "color", "red" );
    return false;
});

    Las llamadas a selectores devuelven el mismo conjunto jQuery

    Metiendo y sacando cosas

    Aparte de html, que mete código a cascoporro, tenemos

    • after: añade código después del elemento (siguiente en el DOM) y append después del contenido del elemento.
    • prepend lo añade antes del contenido del elemento.
    • remove elimina un elemento, mientras que clone lo copia.

    Más texto por todos lados

    (^_^) 

    $("#texto form").submit(function(){
    $("#texto p").prepend( "Mira: ")
      .append(" - ¡Qué feliz!")
      .after('<p>Este no: |-)</p>');
    return false;
});

    También nos lo podemos cargar

    (^_^) 

    $("#clona").submit(function(){
    $("#texto_2 p").after( $("#texto_2 p").clone());
    return false;
});
$("#borra").submit(function(){
    $("#texto_2 p").remove(); return false;
});

    ¿Cómo funciona clone?

    • Crea un nuevo elemento, pero desligado del resto del DOM.
    • Por eso tenemos que usar after para indicarle dónde colocarlo.

    Podemos ser más selectivos

    (^_^) 

    $("#clona_2").submit(function(){
$("#texto_3 p").after( $("#texto_2 p").clone()
  .addClass("nuevo").css("color","green")); return false;
  });
$("#borra_2").submit(function(){
  $("#texto_3 p.nuevo").remove(); return false;
});

    Clonando con clase

    • addClass añade un atributo de clase al elemento que se crea
    • Ese atributo de clase se usa para seleccionar el elemento (o elementos) que queremos borrar.
    • ¿Qué pasaría si se pulsa varias veces el botón Clona? ¿Por qué?

    Siempre hay más de una forma de hacerlo

    (^_^) 

    $("#clona_3").submit(function(){
      $("#texto_4 p")
        .after( $("#texto_4 p").clone()
        .attr("class","nuevo").css( { "color" : "DarkBlue",
                                      "font-weight": "Bold" }));
      return false;
}); // Resto igual

    Animando el cotarro

    • Igual que JavaScript en su tiempo se usó principalmente para animaciones en el cliente, JQuery también tiene una gran cantidad de transiciones y animaciones .
    • Usa las transiciones de CSS3 si puede ser.
    • Cada animación tiene sus propias opciones.

    Mostrando lo oculto

    ♀ ♂

    $("#hembra").mouseover( function( event ){
			    $("#macho").toggle();
      });

    Explicando lo sucedido

    toggle cambia el atributo display pasando de oculto a visible y viceversa.

    Imagen de Trois Têtes (TT)

    Tampoco hay que marginar: animate

    ♀ ♂

    $("#hembra_2").hover( function( event ){
			    $("#macho_2")
      .animate( {marginBottom: 10, marginLeft: 10});
},function(event) {
    $("#macho_2")
      .animate( {marginBottom: 0, marginLeft: 0}); 
});

    Explicando lo animado

    • hover es equivalente a onMouseIn y onMouseOut, por lo que usa dos callbacks.
    • animate recibe como parámetro usa las propiedades CSS que se van a cambiar.

    Deslizándose despacito

    Las claves de la animación

    • slideDown y slideUp deslizan en sendas direcciones lo que estaba previamente oculto.
    • fadeIn y fadeOut van cambiando la transparencia con cierta cadencia hasta aparecer o desaparecer..

    Navegando por un mar de objetos

    • Partiendo de los selectores anteriores, jQuery permite trabajar con los nodos del DOM adyacentes.
    • parent() accede al nodo padre, next() y prev() al siguiente y anterior, children() a todos los hijos y .siblings() a los hermanos.

    Pero poca cosa se puede hacer sólo en el cliente

    • La mayoría de las aplicaciones trabajan con un servidor, aunque sea solamente para comprobar actualizaciones o servir publicidad.
    • JS es hoy en día el único lenguaje que permite escribir aplicaciones cliente-servidor sin cambiar de lenguaje
    • El servidor puede usar CouchDB (una base de datos NoSQL) o node.js, un intérprete similar en concepto a JQuery

    Sirviendo el refranero

    • Creamos una aplicación en node.js usando express, la librería para APIs tipo REST.
    • La aplicación sirve las páginas estáticas
    • Cuando se hace una petición GET a /refranes envía código JSON con un refrán aleatorio.
    • Podemos probarla (si la tenemos instalada, claro)

	$.ajax({

    Explicando héroes mitológicos: Ajax

    • .ajax() es la función genérica de JQuery para llamar al servidor.
    • Primera precaución: solo se puede llamar al servidor del que haya salido la página. Por eso el servidor JS envía también la página HTML.
    • Los dos parámetros imprescindibles son url, la dirección a la que se va a acceder y success, la función callback a la que se llama cuando la respuesta tiene éxito.

    Trabajando en móviles con jQuery Mobile

    • jQuery mobile usa atributos de HTML5 para contruir interfaces táctiles sobre jQuery y jQuery UI.
    • Multiplataforma y basado en markup más que en órdenes (como aprender de nuevo HTML).
    • Accesibilidad integrada.
    • Funciona con jQuery hasta 1.8.3

    Estructurando la aplicación en páginas

    <div data-role="page" id='p1'> <div data-role="header"><h1>Esta aplicación es fantabulosa</h1></div> <div data-role="content"> <p>Y hace un montón de cosas</p> <p> <a href="#p2" data-role="button" data-transition="fade">Pasa página</a> </p> </div> <div data-role="footer"> <h3>Pero puedes pasar a la página siguiente</h3> </div> </div>

    La segunda página

    <div data-role="page" id='p2'> <div data-role="header"><h1>Y aquí empezamos a hacerlo todo</h1></div> <div data-role="content"><p>En realidad no hacemos mucho</p> <p> <a href="#p1" data-role="button" data-transition="turn">Pasa página</a> </p> </div> <div data-role="footer"><h3>Aquí nos quedamos</h3></div> </div>

    Datos por encima de nuestras posibilidades

    • jQuery-Mobile usa data- (de HTML5) para estructurar la aplicación; también accesibles por API.
    • Cada div con el data-role=page actúa como una página
    • data-role indica el significado del elemento y ayuda a su presentación.
    • data-transition codifica las transiciones de página.
